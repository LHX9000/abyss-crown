# ABYSS CROWN 图形小游戏需求文档

## 1. 项目摘要

本文件是 `ABYSS CROWN` 项目的需求文档，归档位置为 `docs/REQUIREMENTS.md`。内容已经从原来的“终端文字冒险项目规划”调整为“SDL2 图形小游戏需求”。

`ABYSS CROWN` 是一个使用 C++17 开发的 2D 俯视角像素地牢小游戏。玩家在暗色地牢地图中移动角色、探索房间、收集 3 个记忆碎片，并在生命值耗尽前找到出口。

项目使用 SDL2 作为图形库，让游戏具备市面小游戏的基本观感：独立窗口、角色、地图、道具、出口、状态栏、提示面板、开始界面、胜利界面和失败界面。画面不要求精细美术，可以先用色块、简单几何图形、低成本贴图和基础动画完成，但最终效果不能只是控制台程序。

AI 部分采用“默认规则 AI + 可选 llama.cpp 增强”的方式。没有本地模型时，游戏仍然可以完整运行；配置 llama.cpp 和 GGUF 模型后，AI 可以生成更自然的事件提示、氛围描述或 NPC 文本。

## 2. 项目目标

### 2.1 产品目标

- 做出一个可运行、可演示的 C++ 图形小游戏。
- 游戏外观看起来像一个完整小游戏，而不是命令行练习程序。
- 通过 SDL2 展示基础游戏开发能力，包括窗口、渲染、输入、碰撞、UI 和游戏状态切换。
- 通过规则 AI 和可选 llama.cpp 展示 AI 在小游戏中的实践方式。

### 2.2 玩家目标

玩家需要：

- 在地牢地图中探索。
- 避开危险区域或事件。
- 收集 3 个记忆碎片。
- 根据 AI 提示理解当前处境。
- 找到出口并逃离地牢。

### 2.3 技术目标

- 使用 C++17。
- 使用 SDL2 创建窗口、处理输入和渲染图形。
- 使用 `Makefile` 管理编译、运行和清理命令。
- llama.cpp 作为可选增强，不作为强制依赖。
- 没有 llama.cpp、没有模型文件时，游戏仍然可以完整通关。

## 3. C++、SDL2 与 VS Code 配置需求

### 3.1 VS Code 扩展

建议在 VS Code 中安装：

- `C/C++`：提供 C++ 语法提示、代码补全、跳转定义等能力。
- `CodeLLDB`：用于 macOS 上调试 C++ 程序。

### 3.2 C++ 编译器

当前电脑已经检测到 Apple clang++，可以直接编译 C++17 程序。

检查命令：

```bash
clang++ --version
```

如果能看到版本信息，说明 C++ 编译器可用。

### 3.3 make

本项目继续使用 `Makefile`，因为当前电脑已经具备 `make`，且项目规模适合轻量构建方式。

检查命令：

```bash
make --version
```

后续项目完成后，常用命令为：

```bash
make
```

编译游戏。

```bash
make run
```

运行游戏。

```bash
make clean
```

清理编译产物。

### 3.4 SDL2

项目需要 SDL2 图形库。

macOS 上推荐使用 Homebrew 安装：

```bash
brew install sdl2
```

如果后续需要在窗口中绘制中文或更完整的文字 UI，可以再增加 `SDL_ttf`：

```bash
brew install sdl2_ttf
```

第一版默认只强制要求 SDL2。若字体配置复杂，核心状态可以先通过简单 UI 图形、英文短文本或窗口标题展示；后续再增强为完整字体渲染。

### 3.5 llama.cpp 可选配置

本项目不强制安装 llama.cpp，也不强制下载模型。

如果后续需要启用本地大模型，可以：

- 安装 llama.cpp。
- 下载 GGUF 格式模型。
- 在 `config/game_config.txt` 中填写 llama.cpp 可执行文件路径和模型路径。
- 游戏检测到配置可用时，调用 llama.cpp 生成 AI 文本。
- 如果配置缺失或调用失败，自动回退到内置规则 AI。

## 4. 功能需求

### 4.1 游戏流程

游戏需要包含以下状态：

- 主界面：显示游戏名称、开始提示和退出提示。
- 游戏中：玩家控制角色探索地图。
- 胜利界面：玩家收集 3 个记忆碎片并到达出口后显示。
- 失败界面：玩家生命值降为 0 后显示。

### 4.2 玩家控制

玩家通过键盘操作：

- `WASD` 或方向键：移动角色。
- `Space` 或 `Enter`：交互，例如拾取碎片、触发机关、确认提示。
- `Esc`：退出当前游戏或关闭窗口。

### 4.3 地图与探索

地图采用 2D 俯视角。

基础地图元素包括：

- 地板：玩家可以移动。
- 墙体：阻挡玩家移动。
- 记忆碎片：玩家可以拾取。
- 出口：满足条件后可通关。
- 危险区域或事件点：可能扣除生命值或触发 AI 提示。

第一版地图可以是固定地图，不要求随机生成。

### 4.4 碰撞与交互

游戏需要实现基础碰撞：

- 玩家不能穿过墙体。
- 玩家接触记忆碎片后可以拾取。
- 玩家接触出口时，如果碎片数量不足，显示提示；如果碎片数量达到 3，进入胜利界面。
- 玩家接触危险区域时扣除生命值，并触发提示。

### 4.5 玩家状态

玩家状态至少包括：

- 当前生命值。
- 已收集记忆碎片数量。
- 当前位置。
- 当前 AI 提示文本。

### 4.6 胜负条件

胜利条件：

- 玩家收集 3 个记忆碎片。
- 玩家到达出口。

失败条件：

- 玩家生命值降为 0。

主动退出：

- 玩家按 `Esc` 或关闭窗口时，游戏正常退出。

## 5. 视觉需求

### 5.1 整体风格

画面风格定为“暗色地牢 + 高亮道具 + 简单 UI 面板”。

不要求精细美术，但至少要做到：

- 有独立游戏窗口。
- 有可见地图区域。
- 有可移动角色。
- 有墙体、地板、道具和出口。
- 有血量和碎片数量显示。
- 有 AI 提示区域。
- 有开始、胜利、失败界面。

### 5.2 窗口尺寸

建议窗口尺寸：

- 宽度：`960`
- 高度：`640`

布局建议：

- 左侧或中央为地图区域。
- 顶部或右侧为状态 UI。
- 底部为 AI 提示面板。

### 5.3 图形表现

第一版可以使用 SDL2 原生绘制：

- 矩形表示地图格子。
- 圆形或高亮矩形表示玩家。
- 发光色块表示记忆碎片。
- 特殊颜色区域表示出口。
- 红色或紫色区域表示危险点。

如果后续增加资源，可以在 `assets/` 中放置图片、字体或音效。

### 5.4 视觉反馈

游戏需要有基础视觉反馈：

- 玩家移动时位置变化明确。
- 拾取碎片后碎片从地图上消失。
- 生命值变化能在 UI 中体现。
- 胜利和失败界面与游戏中界面明显不同。
- AI 提示更新后能在提示面板中显示。

## 6. AI 需求

### 6.1 AiDirector

AI 模块命名为 `AiDirector`，含义是“AI 导演”。

它负责：

- 根据玩家状态生成提示。
- 根据当前区域生成氛围描述。
- 在玩家触发事件时生成反馈。
- 在 llama.cpp 可用时调用本地模型生成文本。

### 6.2 默认规则 AI

默认规则 AI 必须可用。

规则 AI 根据以下信息生成提示：

- 玩家生命值。
- 已收集碎片数量。
- 玩家是否接近出口。
- 玩家是否触发危险区域。
- 玩家是否长时间没有收集到新碎片。

这样做的原因是：即使没有本地大模型，项目也能稳定演示 AI 提示机制。

### 6.3 llama.cpp 增强 AI

llama.cpp 是可选增强。

启用后，游戏将当前状态整理成 prompt，例如：

- 当前房间或区域。
- 玩家生命值。
- 已收集碎片数量。
- 最近触发的事件。

llama.cpp 返回文本后，游戏把结果显示在窗口中的 AI 提示面板。

如果 llama.cpp 路径错误、模型不存在、调用超时或返回为空，游戏自动回退到规则 AI，不能崩溃。

## 7. 非功能需求

- 项目必须能在 VS Code 中打开和编辑。
- 构建命令必须简单，优先使用 `make`。
- 项目结构要清晰，方便理解 C++ 游戏项目如何拆分模块。
- 不硬编码密钥、Token 或敏感信息。
- 不依赖网络才能运行基础游戏。
- 不要求大型剧情系统、复杂物理、联网或精细美术。
- 第一版重点是可运行、完整、有图形界面、有 AI 机制。

## 8. 项目目录框架需求

计划目录结构如下：

```text
cpp_llama_adventure/
├── Makefile
├── README.md
├── docs/
│   ├── REQUIREMENTS.md
│   ├── DESIGN.md
│   └── TEST_PLAN.md
├── assets/
│   ├── fonts/
│   ├── images/
│   └── sounds/
├── config/
│   └── game_config.txt
├── include/
│   ├── AiDirector.h
│   ├── Game.h
│   ├── Input.h
│   ├── LlamaDirector.h
│   ├── Player.h
│   ├── Renderer.h
│   ├── RuleBasedDirector.h
│   ├── Scene.h
│   └── World.h
└── src/
    ├── AiDirector.cpp
    ├── Game.cpp
    ├── Input.cpp
    ├── LlamaDirector.cpp
    ├── Player.cpp
    ├── Renderer.cpp
    ├── RuleBasedDirector.cpp
    ├── Scene.cpp
    ├── World.cpp
    └── main.cpp
```

各部分用途：

- `Makefile`：统一管理编译、运行、清理命令。
- `README.md`：写给使用者看的项目说明，包括安装依赖、运行方式和 llama.cpp 可选配置。
- `docs/REQUIREMENTS.md`：当前需求文档。
- `assets/`：后续存放字体、图片、音效等资源；第一版可以为空或只放占位说明。
- `config/`：保存游戏配置，例如是否启用 llama.cpp、模型路径、难度等。
- `include/`：保存 C++ 头文件，声明类和接口。
- `src/`：保存 C++ 源文件，实现具体逻辑。

## 9. 实现顺序需求

### 第一步：建立 SDL2 最小窗口

- 创建 SDL2 窗口。
- 创建渲染器。
- 设置基础事件循环。
- 支持关闭窗口和 `Esc` 退出。

目标是先证明 SDL2 环境可用。

### 第二步：实现游戏状态切换

- 主界面。
- 游戏中。
- 胜利界面。
- 失败界面。

### 第三步：实现地图和玩家

- 绘制固定地图。
- 绘制玩家。
- 支持 `WASD` 或方向键移动。
- 实现墙体碰撞。

### 第四步：实现道具、出口和生命值

- 添加 3 个记忆碎片。
- 添加出口。
- 添加危险区域。
- 添加生命值和碎片数量 UI。

### 第五步：实现规则 AI 提示

- 根据玩家状态生成提示文本。
- 在提示面板中显示。
- 玩家拾取、受伤、接近出口时更新提示。

### 第六步：实现 llama.cpp 可选接入

- 读取 `config/game_config.txt`。
- 判断是否启用 llama.cpp。
- 调用本地 llama.cpp。
- 失败时自动回退到规则 AI。

### 第七步：完善文档与演示

- 完善 `README.md`。
- 写清楚 SDL2 安装步骤。
- 写清楚 llama.cpp 是可选增强。
- 做完整运行和通关测试。

## 11. 默认实现口径

- 游戏采用俯视角 2D 小地图。
- 不做复杂物理系统。
- 不做联网。
- 不做大型剧情分支。
- 第一版可以用 SDL2 原生绘制矩形和简单 UI。
- 若中文字体渲染成本过高，第一版 UI 文本可以使用英文短文本，README 和需求文档继续使用中文。
- llama.cpp 不强制安装；没有模型时，游戏仍能完整通关。
- 视觉目标是“低成本但完整”，不是精细商业美术。

## 12. 验收标准

项目实现完成后，需要满足：

- 使用 SDL2 打开图形窗口。
- 有主界面、游戏中界面、胜利界面、失败界面。
- 有可移动角色、地图、墙体、记忆碎片、出口和危险区域。
- 有生命值、碎片数量和 AI 提示显示。
- 玩家可以收集 3 个记忆碎片并通关。

- 没有 llama.cpp 时仍能运行。
- llama.cpp 配置失败时不会导致游戏崩溃。
- 文档说明清楚 C++、SDL2、VS Code、make 和 llama.cpp 可选配置。
