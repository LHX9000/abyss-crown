# ABYSS CROWN 图形小游戏设计文档

## 1. 设计目标

`ABYSS CROWN` 是一个使用 C++17 和 SDL2 开发的 2D 俯视角像素地牢小游戏。玩家在暗色地牢中移动角色、收集 3 个记忆碎片、避开危险区域，并在生命值耗尽前找到出口。

本设计的重点是做出一个“低成本但完整”的小游戏：画面不追求精细美术，但必须具备游戏窗口、地图、角色、道具、UI 状态、提示面板、开始界面、胜利界面和失败界面。AI 部分采用默认规则 AI，并预留 llama.cpp 可选接入能力。

## 2. 总体架构

项目采用模块化结构，核心分为以下层次：

- 游戏控制层：负责初始化、主循环、场景切换和退出。
- 输入层：负责把键盘事件转换成游戏动作。
- 世界与状态层：负责地图、玩家、道具、出口、危险区域和胜负条件。
- 渲染层：负责使用 SDL2 绘制地图、角色、UI 和场景界面。
- AI 层：负责生成提示文本，默认使用规则 AI，可选调用 llama.cpp。
- 配置层：负责读取游戏配置，例如是否启用 llama.cpp、模型路径和难度设置。

整体流程是：`main` 创建 `Game`，`Game` 初始化 SDL2、世界、玩家、渲染器和 AI 导演，然后进入主循环。主循环不断处理输入、更新游戏状态、生成提示并渲染画面。

## 3. 目录与模块设计

计划目录结构如下：

```text
cpp_llama_adventure/
├── Makefile
├── README.md
├── assets/
├── config/
│   └── game_config.txt
├── docs/
│   ├── REQUIREMENTS.md
│   ├── DESIGN.md
│   └── TEST_PLAN.md
├── include/
└── src/
```

主要模块设计：

- `Game`：游戏总控制器，负责初始化、主循环、状态切换和资源释放。
- `Input`：输入处理模块，负责识别 `WASD`、方向键、`Space`、`Enter` 和 `Esc`。
- `Scene`：场景状态模块，管理主界面、游戏中、胜利界面和失败界面。
- `World`：地图模块，保存固定地图、格子类型、碰撞规则和交互点。
- `Player`：玩家模块，保存位置、生命值、碎片数量和受伤/拾取逻辑。
- `Renderer`：渲染模块，封装 SDL2 绘制逻辑。
- `AiDirector`：AI 提示统一接口。
- `RuleBasedDirector`：默认规则 AI，根据玩家状态生成提示。
- `LlamaDirector`：可选 llama.cpp 适配器，调用失败时回退到规则 AI。

这样拆分的原因是：游戏逻辑、图形绘制和 AI 提示互相独立，后续改画面或改 AI 时不会牵动全部代码。

## 4. 核心流程设计

### 4.1 启动流程

1. 程序从 `main` 启动。
2. 创建 `Game` 实例。
3. 初始化 SDL2、窗口和渲染器。
4. 加载配置文件。
5. 初始化地图、玩家和 AI 导演。
6. 进入主界面。

### 4.2 主循环流程

每一帧执行：

1. 读取 SDL2 事件。
2. 将键盘输入转换为游戏动作。
3. 根据当前场景处理动作。
4. 更新玩家、地图和胜负状态。
5. 必要时更新 AI 提示。
6. 调用 `Renderer` 绘制当前场景。
7. 控制帧率，保持画面稳定。

### 4.3 移动与碰撞流程

1. 玩家按下移动键。
2. `Input` 生成移动方向。
3. `Game` 计算目标位置。
4. `World` 判断目标位置是否为墙体。
5. 如果不是墙体，更新玩家位置。
6. 检查当前位置是否包含碎片、出口或危险区域。

### 4.4 拾取与通关流程

1. 玩家移动到碎片位置。
2. `World` 标记该碎片已被拾取。
3. `Player` 的碎片数量加 1。
4. AI 提示更新为拾取反馈。
5. 玩家移动到出口时，如果碎片数量达到 3，则切换到胜利界面；否则显示“还需要碎片”的提示。

### 4.5 失败流程

1. 玩家触发危险区域。
2. `Player` 生命值减少。
3. UI 立即更新生命值。
4. 如果生命值降为 0，场景切换到失败界面。

## 5. 数据结构设计

### 5.1 地图数据

地图第一版使用固定二维格子，格子类型包括：

- `Floor`：可通行地板。
- `Wall`：不可通行墙体。
- `Shard`：记忆碎片。
- `Exit`：出口。
- `Danger`：危险区域。

每个格子对应一个固定大小的 SDL2 绘制区域，例如 `32x32` 或 `40x40` 像素。

### 5.2 玩家数据

玩家需要保存：

- `position`：当前地图坐标。
- `health`：当前生命值，初始值建议为 3。
- `shardCount`：已收集碎片数量。
- `isAlive`：是否仍可继续游戏。

### 5.3 游戏状态数据

游戏状态包括：

- 当前场景。
- 当前 AI 提示文本。
- 是否启用 llama.cpp。
- 是否正在运行。
- 最近触发的事件类型。

## 6. 渲染与界面设计

窗口建议尺寸为 `960x640`。

界面布局：

- 中央或左侧为地图区域。
- 顶部或右侧为状态栏，显示生命值和碎片数量。
- 底部为 AI 提示面板。

第一版图形可以全部使用 SDL2 原生绘制：

- 深色矩形表示地板。
- 更深或带边框的矩形表示墙体。
- 高亮矩形或圆形表示玩家。
- 蓝色或金色发光色块表示记忆碎片。
- 绿色区域表示出口。
- 红色或紫色区域表示危险点。

主界面、胜利界面和失败界面需要和游戏中界面明显区分。若第一版不接入 `SDL_ttf`，可以通过大面积色块、窗口标题、简单按钮区域和 README 操作说明完成基础表达；后续再加入字体渲染。

## 7. AI 设计

### 7.1 AI 接口

`AiDirector` 提供统一提示接口。游戏逻辑只关心“给定当前状态，返回一段提示文本”，不直接关心提示来自规则还是 llama.cpp。

### 7.2 规则 AI

`RuleBasedDirector` 是默认实现，始终可用。它根据以下信息生成提示：

- 玩家生命值。
- 已收集碎片数量。
- 玩家是否接近出口。
- 玩家是否进入危险区域。
- 玩家是否刚拾取碎片。

规则 AI 的作用是保证项目没有本地模型时也能演示 AI 提示机制。

### 7.3 llama.cpp 可选增强

`LlamaDirector` 在配置可用时调用 llama.cpp。调用输入包括当前区域、生命值、碎片数量和最近事件。模型返回的文本显示在 AI 提示面板。

如果 llama.cpp 路径错误、模型不存在、调用超时或返回为空，游戏自动回退到 `RuleBasedDirector`，不能影响主流程。

## 8. 配置与构建设计

项目使用 `Makefile`，提供：

- `make`：编译游戏。
- `make run`：运行游戏。
- `make clean`：清理编译产物。

配置文件为 `config/game_config.txt`，用于保存：

- 是否启用 llama.cpp。
- llama.cpp 可执行文件路径。
- GGUF 模型文件路径。
- 游戏难度或调试开关。

第一版强制依赖 SDL2，不强制依赖 `SDL_ttf` 和 llama.cpp。

跨平台交付同时使用 CMake：

- `CMakeLists.txt` 作为 CLion 和 GitHub Actions 的统一构建入口。
- `CMakePresets.json` 提供 Windows MinGW Debug/Release 配置。
- `vcpkg.json` 声明 SDL2 和 SDL2_ttf 依赖。
- 构建完成后自动复制 `assets/` 和 `config/` 到可执行文件目录。
- 游戏通过统一资源路径模块，以可执行文件目录为基准查找资源和配置。
- Windows 默认使用规则 AI；llama.cpp 仍为可选功能。

## 9. 异常处理与降级策略

- SDL2 初始化失败：输出错误信息并退出程序。
- 窗口或渲染器创建失败：释放已创建资源并退出。
- 配置文件不存在：使用默认配置继续运行。
- llama.cpp 配置缺失：使用规则 AI。
- llama.cpp 调用失败：使用规则 AI。
- 外部资源缺失：使用 SDL2 几何图形绘制基础画面。

这样设计的目标是让基础游戏稳定可运行，避免因为可选能力失败导致整个项目无法演示。

## 10. 测试设计

后续测试文档放在 `docs/TEST_PLAN.md`。

需要覆盖：

- 编译测试：确认 `make` 能生成可执行文件。
- SDL2 窗口测试：确认 `make run` 能打开窗口并正常关闭。
- 输入测试：确认 `WASD`、方向键、`Space`、`Enter` 和 `Esc` 可用。
- 碰撞测试：确认玩家不能穿墙。
- 拾取测试：确认碎片能被拾取并从地图消失。
- 胜利测试：确认收集 3 个碎片后到达出口进入胜利界面。
- 失败测试：确认生命值为 0 后进入失败界面。
- AI 回退测试：确认没有 llama.cpp 时规则 AI 正常工作。

## 11. 后续扩展方向

- 增加 `SDL_ttf`，让窗口内显示更完整的文字。
- 增加图片资源，让角色、墙体和道具更像正式游戏素材。
- 增加音效和背景音乐。
- 增加随机地图或多关卡。
- 增加更多 AI 事件，例如动态 NPC 提示、谜题提示和剧情旁白。
